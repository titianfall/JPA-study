## 1. 프로젝트 생성

[스프링 부트 스타터](https://start.spring.io/)에서 생성한다.

| 항목 | 값 |
|------|-----|
| Project | **Gradle - Groovy** |
| Spring Boot | **3.x.x 최신 버전** (2.x는 지원 종료로 선택 불가) |
| Java | **17 또는 21** |
| groupId | `jpabook` |
| artifactId | `jpashop` |
| Dependencies | web, thymeleaf, jpa, h2, lombok, **validation** |

> ⚠️ **Validation 모듈은 영상에 없지만 꼭 추가**해야 한다. (JSR-303 validation with Hibernate validator)

### ⚠️ 강의 영상(부트 2.x)과 달라진 점

- 스프링 부트 버전: `2.1.x` → `3.x.x`
- 자바 버전: `1.8` → `17`
- `validation` 모듈 추가됨

### build.gradle 전체 설정

```groovy
plugins {
	id 'java'
	id 'org.springframework.boot' version '3.4.4'
	id 'io.spring.dependency-management' version '1.1.7'
}

group = 'jpabook'
version = '0.0.1-SNAPSHOT'

java {
	toolchain {
		languageVersion = JavaLanguageVersion.of(17)
	}
}

configurations {
	compileOnly {
		extendsFrom annotationProcessor
	}
}

repositories {
	mavenCentral()
}

dependencies {
	implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
	implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'
	implementation 'org.springframework.boot:spring-boot-starter-validation'
	implementation 'org.springframework.boot:spring-boot-starter-web'
	compileOnly 'org.projectlombok:lombok'
	runtimeOnly 'com.h2database:h2'
	annotationProcessor 'org.projectlombok:lombok'
	testImplementation 'org.springframework.boot:spring-boot-starter-test'
	testRuntimeOnly 'org.junit.platform:junit-platform-launcher'
}

tasks.named('test') {
	useJUnitPlatform()
}
```

### 동작 확인

- 기본 테스트 케이스 실행
- 스프링 부트 메인 실행 후 에러 페이지로 간단히 동작 확인 (`http://localhost:8080`)

### 롬복(Lombok) 적용 (IntelliJ)

1. Preferences → plugin → **lombok** 검색 후 설치 (재시작)
2. Preferences → Annotation Processors 검색 → **Enable annotation processing** 체크 (재시작)
3. 임의의 테스트 클래스에서 `@Getter`, `@Setter` 동작 확인

### ⚠️ IntelliJ 빌드 설정 — 반드시 Gradle 선택

스프링 부트 3.2부터 **JDK 17 이상** + 빌드 도구로 **Gradle**을 선택해야 한다.

1. **프로젝트 JDK**: `File → Project Structure`(Ctrl+Alt+Shift+S) → JDK를 자바 17로 지정
2. **Gradle 설정**: `File → Settings`(Ctrl+Alt+S) → `Build, Execution, Deployment → Build Tools → Gradle`
   - `Build and run using`: **Gradle**
   - `Build tests using`: **Gradle**
   - `Gradle JVM`: 자바 17 이상

> ⚠️ IntelliJ IDEA로 선택하면 이후에 문제가 될 수 있다. 반드시 **Gradle**로!

## 3. View 환경 설정 (Thymeleaf)

스프링 부트의 thymeleaf viewName 매핑 규칙:

```
resources/templates/ + {ViewName} + .html
```

### 컨트롤러 — `HelloController`

```java
package jpabook.jpashop;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class HelloController {

    @GetMapping("hello")
    public String hello(Model model) {
        model.addAttribute("data", "hello!!");
        return "hello";   // → resources/templates/hello.html
    }
}
```

### 템플릿 — `resources/templates/hello.html`

```html
<!DOCTYPE HTML>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Hello</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
</head>
<body>
<p th:text="'안녕하세요. ' + ${data}">안녕하세요. 손님</p>
</body>
</html>
```

### 정적 페이지 — `resources/static/index.html`

```html
<!DOCTYPE HTML>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Hello</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
</head>
<body>
Hello
<a href="/hello">hello</a>
</body>
</html>
```

> 💡 `spring-boot-devtools` 라이브러리를 추가하면 html 파일을 **컴파일만 해도**(메뉴 build → Recompile) 서버 재시작 없이 View 파일 변경이 반영된다.

## 4. H2 데이터베이스 설치

개발·테스트 용도로 가볍고 편리한 DB, 웹 화면 제공. — https://www.h2database.com

- 스프링 부트 3.x → **2.1.214 버전 이상** 사용 (부트 2.x는 1.4.200)

### 데이터베이스 파일 생성 순서

1. 최초 한 번은 `jdbc:h2:~/jpashop` 으로 접속 → `~/jpashop.mv.db` 파일 생성 확인
2. 이후부터는 `jdbc:h2:tcp://localhost/~/jpashop` 으로 접속

> ⚠️ **MVCC 옵션 제거**: H2 1.4.198부터 MVCC 옵션이 제거되어, 영상처럼 `;MVCC=TRUE`를 붙이면 오류가 발생한다.
> - 영상: `jdbc:h2:tcp://localhost/~/jpashop;MVCC=TRUE`
> - 수정: `jdbc:h2:tcp://localhost/~/jpashop`

## 5. JPA와 DB 설정, 동작 확인

### `main/resources/application.yml`

```yaml
spring: #띄어쓰기 없음
  datasource: #띄어쓰기 2칸
    url: jdbc:h2:tcp://localhost/~/jpashop #4칸
    username: sa
    password:
    driver-class-name: org.h2.Driver

  jpa: #띄어쓰기 2칸
    hibernate: #띄어쓰기 4칸
      ddl-auto: create #띄어쓰기 6칸
    properties: #띄어쓰기 4칸
      hibernate: #띄어쓰기 6칸
        # show_sql: true #띄어쓰기 8칸
        format_sql: true #띄어쓰기 8칸

logging.level: #띄어쓰기 없음
  org.hibernate.SQL: debug #띄어쓰기 2칸
  # org.hibernate.orm.jdbc.bind: trace #스프링 부트 3.x, hibernate6
```

- `ddl-auto: create` — 애플리케이션 실행 시점에 테이블을 **drop 후 재생성**
- `show_sql` — `System.out`에 SQL 출력 (사용 지양)
- `org.hibernate.SQL: debug` — **로거를 통해** SQL 출력 (권장)

> ⚠️ yml 파일은 **띄어쓰기 2칸**으로 계층을 만든다. 띄어쓰기가 틀리면 설정이 적용되지 않는다.

> 💡 모든 로그 출력은 가급적 로거를 통해 남기자.

### 회원 엔티티

```java
package jpabook.jpashop;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.Id;
import lombok.Getter;
import lombok.Setter;

@Entity
@Getter @Setter
public class Member {

    @Id @GeneratedValue
    private Long id;
    private String username;
}
```

> ⚠️ **[영상과 다른 점] javax → jakarta**: 스프링 부트 3.x부터 `javax.persistence` 대신 `jakarta.persistence`를 사용한다. (기본편 실습과 동일한 변경점)

### 회원 리포지토리

```java
package jpabook.jpashop;

import jakarta.persistence.EntityManager;
import jakarta.persistence.PersistenceContext;
import org.springframework.stereotype.Repository;

@Repository
public class MemberRepository {

    @PersistenceContext
    EntityManager em;

    public Long save(Member member) {
        em.persist(member);
        return member.getId();
    }

    public Member find(Long id) {
        return em.find(Member.class, id);
    }
}
```

> IntelliJ에서 테스트 파일은 대상 클래스에서 `Cmd+Shift+T`(macOS) 또는 `Ctrl+Shift+T`(Windows/Linux)로 만들 수 있다.

#### `save()`가 `Member`가 아닌 ID를 반환하는 이유

저장 명령은 엔티티 자체가 아니라 식별자만 반환한다. 이는 **명령(Command)과 조회(Query)를 분리**하는 CQS(Command-Query Separation) 방식이다. 호출부에서 반환된 관리 상태 엔티티를 계속 수정하면 같은 영속성 컨텍스트 안에서 의도하지 않은 `UPDATE`가 발생할 수 있다.

따라서 저장한 뒤 조회가 필요하면 반환받은 ID로 `find()`를 호출한다. 호출부는 저장 결과를 식별자로만 다루고, 엔티티 사용이 필요한 경우에만 명시적으로 조회하게 된다.

### 테스트

```java
package jpabook.jpashop;

import jakarta.transaction.Transactional;
import org.assertj.core.api.Assertions;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.annotation.Rollback;

@SpringBootTest
public class MemberRepositoryTest {

    @Autowired MemberRepository memberRepository;

    @Test
    @Transactional
    @Rollback(false)
    public void testMember() {
        Member member = new Member();
        member.setUsername("memberA");

        Long savedId = memberRepository.save(member);
        Member findMember = memberRepository.find(savedId);

        Assertions.assertThat(findMember.getId()).isEqualTo(member.getId());
        Assertions.assertThat(findMember.getUsername()).isEqualTo(member.getUsername());
        Assertions.assertThat(findMember).isEqualTo(member); // JPA 엔티티 동일성 보장
    }
}
```

### ⚠️ [영상과 다른 점] 테스트 관련

| 항목 | 영상 (부트 2.x / JUnit4) | 수정 (부트 3.x / JUnit5) |
|------|--------------------------|--------------------------|
| Test 애노테이션 | `org.junit.Test` | `org.junit.jupiter.api.Test` |
| Runner | `@RunWith(SpringRunner.class)` 필요 | `@RunWith` **사용 안 함** (제거) |

> 💡 스프링 부트 덕분에 복잡한 설정이 대부분 자동화되었다. `persistence.xml`도, `LocalContainerEntityManagerFactoryBean`도 없다. (순수 스프링 + JPA 설정은 자바 ORM 표준 JPA 프로그래밍 책 참고)

## 6. 쿼리 파라미터 로그 남기기

### 방법 1 — 하이버네이트 로거

스프링 부트 3.x(hibernate6)는 영상(부트 2.x)과 설정 키가 다르다.

```yaml
logging.level:
  # 스프링 부트 2.x, hibernate5
  # org.hibernate.type: trace
  # 스프링 부트 3.x, hibernate6
  org.hibernate.orm.jdbc.bind: trace
```

### 방법 2 — 외부 라이브러리 (p6spy)

https://github.com/gavlyukovskiy/spring-boot-data-source-decorator

```groovy
implementation 'com.github.gavlyukovskiy:p6spy-spring-boot-starter:1.12.1'
```

> ⚠️ **[영상과 다른 점] p6spy 버전**: 영상은 `1.5.6`, 부트 3.x는 `1.12.1`, 부트 4.x는 `2.0.0` 사용.

> 💡 쿼리 파라미터 로그 라이브러리는 시스템 자원을 사용하므로 개발 단계에서는 자유롭게 써도 되지만, **운영 시스템에 적용하려면 꼭 성능테스트** 후 사용하자.

## ✅ 핵심 요약

1. **프로젝트 생성**: start.spring.io — 부트 3.x + Java 17 + Gradle, `validation` 모듈 꼭 추가.
2. **IntelliJ 설정**: 빌드 도구는 반드시 **Gradle**, 롬복 플러그인 + annotation processing 활성화.
3. **View**: thymeleaf 매핑 규칙은 `resources/templates/{ViewName}.html`.
4. **H2**: 최초 `jdbc:h2:~/jpashop` 으로 파일 생성 후 tcp 접속. `;MVCC=TRUE` 옵션은 제거.
5. **JPA 설정**: `application.yml` — `ddl-auto: create`, SQL은 로거(`org.hibernate.SQL: debug`)로 출력.
6. **부트 3.x 차이**: `jakarta.persistence`, JUnit5(`@RunWith` 제거), 쿼리 파라미터 로그는 `org.hibernate.orm.jdbc.bind: trace`.
