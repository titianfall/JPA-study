# 03. 애플리케이션 구현 준비

> 현재 `study/SpringBootJPA/jpashop` 프로젝트를 기준으로, 회원 도메인 개발 전 필요한 구현 준비 사항을 간단히 정리한다.

---

## 1. 패키지 역할 분리

- `domain`: JPA 엔티티, 값 타입, 상태 enum을 둔다.
- `repository`: 엔티티를 저장하고 조회하는 영속성 접근 코드를 둔다.
- `controller`: HTTP 요청을 받고 화면 또는 응답을 반환한다.

기능이 늘어나도 책임이 섞이지 않도록 웹 계층과 영속성 계층을 분리한다. 회원 기능은 `domain.Member`, `repository.MemberRepository`, 이후 추가할 회원 서비스와 컨트롤러가 각각 역할을 나누어 구현한다.

## 2. 엔티티 기본 원칙

```java
@Entity
@Getter
public class Member {

    @Id @GeneratedValue
    @Column(name = "member_id")
    private Long id;

    private String name;

    @Embedded
    private Address address;
}
```

- 식별자는 `@Id`, `@GeneratedValue`로 지정하고 DB 컬럼명은 필요할 때 명시한다.
- 주소처럼 여러 값이 하나의 의미를 가지는 객체는 `@Embeddable` 값 타입으로 만든다.
- 변경은 무분별한 setter 대신 도메인 의미가 드러나는 메서드로 제한한다.
- 연관관계는 기본적으로 지연 로딩(`FetchType.LAZY`)을 사용한다.

## 3. 개발 환경 확인

현재 프로젝트는 Spring Boot 3.x와 JPA `jakarta.persistence`를 사용한다. H2 TCP 서버를 실행한 뒤 `application.yml`의 `jdbc:h2:tcp://localhost/~/jpashop`으로 접속한다.

학습 단계에서는 `ddl-auto: create`로 엔티티 변경을 스키마에 반영한다. 실제 운영 환경에서는 기존 테이블을 삭제할 수 있으므로 이 옵션을 사용하지 않는다.
